CSS-селекторы и их виды

В CSS-селекторы используются для стилизации HTML элементов на веб-странице. Селекторы позволяют обратиться как к группе элементов, так и к только одному из них. Он находится в начале CSS-правила, до фигурных скобок, и определяет к каким HTML-элементам применятся свойства и значения из правила.

Основные виды селекторов CSS:

Селектор элемента (тега): имя тега { Описание правил стиля }
Селектор по элементу предназначен для выбора элементов по имени тега.
Селектор class: .Имя класса { Описание правил стиля }
Классы применяют, когда необходимо определить стиль для одного или нескольких элементов веб-страницы. Элементы при этом могут быть разными, но иметь частично или полностью похожие элементы оформления. В коде HTML имя класса определяется через атрибут class.
Селектор id: #Имя идентификатора { Описание правил стиля }
Селектор по идентификатору предназначен для выбора элемента по идентификатору (значению атрибута id). При описании идентификатора вначале указывается символ решётки (#), затем идёт имя идентификатора. Оно должно начинаться с латинского символа и может содержать в себе символ дефиса (-) и подчеркивания (_). Использование русских букв в именах идентификатора недопустимо. В отличие от классов идентификаторы должны быть уникальны, иными словами, встречаться в коде документа только один раз.
Универсальный селектор: * { Описание правил стиля }
Универсальные селекторы CSS обозначаются обыкновенной звездочкой «*». Универсальный селектор необходим для того, чтобы задать CSS-свойства сразу всем элементам разметки HTML-страницы.

Взаимоотношения между селекторами. Вложенность

Взаимоотношения между селекторами - это когда между ними в CSS-коде ставят какой-либо знак: пробел, запятую, угловую скобку и т.д. Таким образом, можно выбрать какой-либо элемент на странице более точечно.

CSS селекторы потомков, или контекстные селекторы: E F { Описание правил стиля }
Селектор подразумевает выбор элементов потомков (между селекторами ставят пробел). Здесь E это родительский элемент, а F — дочерний элемент, расположенный в контейнере E. В этом случае стиль будет применяться к элементу F.
Дочерние селекторы: E > F { Описание правил стиля } (родитель > дети)
Дочерние селекторы CSS отличаются от селекторов потомков тем, что под дочерним подразумевается только первый потомок, никакие «внуки» и «правнуки» не попадают в эту категорию.
Соседние селекторы: E + F { Описание правил стиля }
Соседними называются элементы веб-страницы, когда они следуют непосредственно друг за другом в коде документа. Для управления стилем соседних элементов используется символ плюса (+), который устанавливается между двумя селекторами E и F. Стиль при такой записи применяется к элементу F, но только в том случае, когда элементы находятся на одном уровне вложенности, т.е. по отношению друг к другу соседи (сиблинги).
Родственные селекторы: E ~ F { Описание правил стиля }
Предназначен для выбора элементов F, которые расположены после E. Стиль при такой записи применяется к элементу F в том случае, если он имеет того же родителя, что и элемент E и следует после него.
Селекторы групп: Селектор 1, Селектор 2, ... Селектор N { Описание правил стиля }
Селекторы групп позволяют задавать стили сразу нескольким элементам. Селекторы группируются в виде списка тегов, разделенных между собой запятыми. В группу могут входить не только селекторы тегов, но также идентификаторы и классы.

Селекторы атрибутов и значений

Селекторы атрибутов и значений — выбирают элементы по их атрибуту или атрибуту и значению.

Селектор атрибута:[атрибут] { Описание правил стиля }
Селекторы по атрибуту предназначены для выбора элементов по имени атрибута и (или) его значению.

Типы селекторов по атрибуту:

  • [attr] – устанавливает стиль для элемента, если задан специфичный атрибут. Его значение в данном случае не важно;
  • [attr=value] – устанавливает стиль для элемента в том случае, если задано определённое значение специфичного атрибута;
  • [attr^=value] – устанавливает стиль для элемента в том случае, если значение атрибута начинается с указанного текста;
  • [attr|=value] – стиль применяется к элементам, у которых атрибут начинается с указанного значения, после которого идёт дефис;
  • [attr$=value] – устанавливает стиль для элемента в том случае, если значение атрибута оканчивается указанным текстом;
  • [attr*=value] – по указанному атрибуту и значению, которое должно содержать value (без привязки к местоположению);
  • [attr~=value] – по имени атрибута и значению, которое содержит value отделённое от других с помощью пробела.

Селекторы псевдокласcа и псевдоэлемента

Псевдоклассы — особый вид селектора, который уточняет тип или состояние. Обычно это какой-то качественный признак: реакция на наведение курсора, порядок следования и другие. Благодаря псевдоклассам мы можем контролировать динамические параметры селекторов. Правила стиля сработают, когда селектор подходит под дополнительный признак.

Селектор псевдокласса - Селектор:псевдокласс { свойство: значение; }
При написании между селектором и псевдоклассом ставится двоеточие (:). Псевдоклассы дают возможность стилизовать элемент на основе не только отношений в DOM-дереве, но и основываясь на внешних факторах, таких как история посещений (например, :visited), состояние содержимого (вроде :checked у некоторых элементов формы) или позиции курсора мыши (например, :hover определяет, находится ли курсор мыши над элементом).

Псевдоэлементы – это селекторы, которые определяют область элементов, которая изначально отсутствует в дереве документа. Эта область создается искусственно с помощью CSS. Ключевое отличие между ними в том, что псевдоклассы определяют именно состояние элементов, которые уже существуют на странице, а псевдоэлементы создают области (искусственные элементы), которых изначально на веб-странице не было. Но и те и другие отсутствуют в исходном коде документа.

Селектор псевдоэлемента - Селектор::псевдоэлемент { свойство: значение; }
При написании между селектором и псевдоклассом ставится двойное двоеточие (::). Псевдоэлементы применяются на сайте только в оформительских целях, т.е. для добавления к элементам определённых дизайнерских решений. С помощью них не рекомендуется добавлять на сайт важную информацию. Если этих элементов нет в DOM, то к ним нельзя добавить обработчики событий или каким-то другим образом взаимодействовать с ними через JavaScript.

На данный момент существует 5 псевдоэлементов:

  • ::first-line — предназначен для изменения оформления первой линии текста;
  • ::first-letter — предназначен для изменения оформления первой буквы текста;
  • ::before — позволяет добавить контент до элемента;
  • ::after — позволяет добавить контент после элемента;
  • ::selection — отвечает за выделение элемента пользователем.
